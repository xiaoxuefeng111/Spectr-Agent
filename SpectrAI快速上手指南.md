# SpectrAI 快速上手指南

从安装到第一次对话，5 分钟掌握 SpectrAI 核心操作

---

## 1. 安装与启动

### 下载安装

1. 前往下载页面获取对应系统的安装包
2. **Windows**：双击 `.exe` 安装包，按向导完成安装
3. **macOS**：将 `.app` 拖入 Applications 文件夹。如果提示"应用已损坏"，参见 FAQ #3

### 前置依赖

SpectrAI 本身不需要额外依赖，但你选择的 AI 提供者可能需要：

| AI 提供者 | 前置要求 | 安装命令 |
|-----------|----------|----------|
| Claude Code | Node.js 18+ | `npm i -g @anthropic-ai/claude-code` |
| Codex CLI | Node.js 22+ | `npm i -g @openai/codex` |
| Gemini CLI | Node.js 18+ | `npm i -g @anthropic-ai/claude-code` (内置) |
| OpenAI 兼容 API | 无 | 只需 API Key + 接口地址 |

---

## 2. 配置 AI 提供者

打开 SpectrAI → 点击左下角 **设置（齿轮图标）** → 选择 **AI provider** 标签页。

### 方式一：使用内置 CLI 工具（推荐）

以 Claude Code 为例：

1. 确保已全局安装：`npm i -g @anthropic-ai/claude-code`
2. 在终端运行 `claude login` 完成账户登录
3. 回到 SpectrAI，找到 Claude Code 卡片，点击 **Test** 按钮确认连通
4. 如果 Test 失败，手动填写 **Executable Path**（参见 FAQ #4）

### 方式二：使用 API Key（第三方中转）

1. 点击页面底部 **「添加自定义 AI 提供者」**
2. Adapter Type 选择 `OpenAI Compatible API`
3. 填写 API Base URL、API Key、模型名称
4. 保存即可使用

> 💡 提示：CLI 类型的 Provider（Claude Code / Codex 等）通过各自的命令行登录认证，不需要在 SpectrAI 里填 API Key。详见 FAQ #1。

## 3. 新建你的第一个会话

### 步骤说明

1. 点击左侧边栏的 **「+」** 按钮打开新建对话框
2. **会话名称** — 可自定义，默认为当前时间
3. **工作目录** — 选择你的项目文件夹，AI 将在此目录下读写文件、执行命令
4. **初始指令**（可选）— 输入希望 AI 立即执行的任务，留空则进入交互模式
5. **AI 提供者** — 选择刚才配好的 Provider（如 Claude Code）
6. **会话模式** — 初次使用建议选 **普通会话**
7. 点击 **「创建」**，等待 AI 就绪后即可开始对话

> 💡 提示：勾选 **「自动接受权限请求」** 可以让 AI 无需你逐次确认就能执行文件读写和命令。适合信任的项目环境。

## 4. 三种会话模式详解

根据任务复杂度选择合适的模式：

### 普通会话（默认）

你和 AI 一对一对话，最基础也是最常用的模式。

适用场景：
- 编程问答
- 单文件修改
- 代码调试
- 日常开发

### Supervisor（指挥官）

AI 变身"项目经理"，可创建多个子 Agent 并行完成任务。

适用场景：
- 大型重构
- 多文件修改
- 并行分析
- 混合多种 AI

### 自主任务（Mission）

描述目标即可，AI 自动规划、组建团队、执行任务。

适用场景：
- 端到端交付
- 自动团队协作
- 任务拆解分配
- 全程自主执行

### 模式选择速查

| 我想做的事 | 推荐模式 |
|------------|----------|
| 问一个编程问题 / 修一个 bug | **普通会话** |
| 分析多个文件的依赖关系 | **Supervisor** |
| 同时修改前端 + 后端 + 测试 | **Supervisor** |
| "帮我完成整个用户注册功能" | **自主任务** |
| "重构项目的状态管理" | **自主任务** |

## 5. AI 提供者选择建议

不确定该用哪个？参考以下建议：

### 提供者概览

| 提供者 | 特点 |
|--------|------|
| Claude Code | 综合推理最强 |
| Codex CLI | 代码生成专长 |
| Gemini CLI | 长文本理解 |
| OpenCode | 多模型切换 |
| iFlow CLI | 工作流集成 |

### 任务类型推荐

| 任务类型 | 推荐 | 原因 |
|----------|------|------|
| 复杂架构设计、多文件重构 | **Claude Code** | 综合推理能力最强 |
| 写代码、修 bug、加功能 | **Codex CLI** | 代码生成专长 |
| 大文件分析、代码审查 | **Gemini CLI** | 超大上下文窗口 |
| 文档总结、知识梳理 | **Gemini CLI** | 擅长长文本理解 |
| 并行多个子任务 | **混合使用** | 不同 AI 视角互补 |

> 💡 提示：在 Supervisor 模式下，主会话可以给不同的子 Agent 分配不同的 AI 提供者，充分发挥各自优势。
